# CHANGELOG

## [v1.0.0] — 2026-05-21

Финальная версия лабораторной работы.

### Добавлено

#### Задачи №1 — SELECT-запросы

- **task1/task1.py** — запрос фильмов жанров «музыка» и «анимация», выпущенных с 1997 года; читает путь к БД из stdin
- **task1/task2.py** — запрос фильмов, название которых заканчивается на «?»
- **task1/task3.py** — список уникальных годов выхода фильмов, название которых начинается на «Х»; результат без дублей
- **task1/task4.py** — список комедий продолжительностью не менее 60 минут (JOIN movies + genres)
- **task1/task5.py** — уникальные треки исполнителя из music.db в алфавитном порядке; читает путь к БД и имя исполнителя из stdin; использует JOIN Artist → Album → Track

#### Задачи №2 — Модификация данных

- **task2/task1.py** — функция `get_result(name)`: удаление всех фильмов жанра «комедия»
- **task2/task2.py** — функция `get_result(name)`: установка длительности 42 мин для записей с `duration IS NULL`
- **task2/task3.py** — функция `get_result(name)`: удвоение длительности всех фильмов жанра «фантастика»
- **task2/task4.py** — функция `get_result(name)`: удаление фильмов, название которых начинается на «Я» и заканчивается на «а» (LIKE 'Я%а')
- **task2/task5.py** — функция `get_result(name)`: удаление боевиков с длительностью ≥ 90 минут
- **task2/task6.py** — функция `get_result(name)`: обрезка длительности мюзиклов до 100 минут (WHERE duration > 100)
- **task2/task7.py** — функция `get_result(name)`: уменьшение длительности фильмов 1973 года в три раза (division by 3)
- **task2/task8.py** — функция `get_result(name)`: удаление фильмов жанра «фантастика», вышедших до 2000 года, с длительностью более 90 минут
- **task2/task9.py** — интерактивная система регистрации пользователей: бесконечный цикл, проверка уникальности логина/email, подтверждение пароля, хранение в SQLite (файл `users.db`)
- **task2/task10.py** — интерактивная система авторизации: бесконечный цикл, поиск пользователя, проверка пароля; сообщения об ошибках: «Пользователь не найден», «Неверный пароль»; при успехе — «Вход выполнен»

#### Инфраструктура

- **Dockerfile** — образ на базе `python:3.12-slim`, копирует БД и скрипты
- **docker-compose.yml** — сервис `solution`, собирает образ и запускает `run_all.sh`
- **solution/run_all.sh** — последовательный запуск всех 15 задач с тестовыми данными через piped stdin; для задач 2.1–2.8 используются отдельные копии БД (деструктивные операции не затрагивают оригинал)
